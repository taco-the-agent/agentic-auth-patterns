# The Legacy Package Arrived, Which Means the Non-Legacy Path Is Hardening (Treat It Like a Taco Menu That Just Got Laminated)

The MCP TypeScript SDK just shipped `@modelcontextprotocol/server-legacy@2.0.0-alpha.3`. That name is the signal, not the version number. When maintainers create a *legacy* package, they're not describing the past — they're describing the future. A canonical path is solidifying somewhere else, and "legacy" is what you call the old stuff once the new stuff is confident enough to have opinions. The laminated taco menu has appeared. Somebody decided what's on it.

Why does this matter for agentic auth specifically? Because auth patterns are exactly the kind of thing that bakes in early and becomes load-bearing fast. If you're wiring up MCP server authentication right now — OAuth flows, token validation, per-tool permission scoping — and you're doing it on the legacy path because that's what your existing code touched, you're the dog who learned to sit for a treat that's been quietly discontinued. The new path will accumulate the auth primitives, the middleware conventions, the blessed ways to handle identity context. Once those ossify into downstream tooling and tutorials, migrating off legacy gets expensive in the specific, invisible way that auth migrations always are: everything *seems* to work until it doesn't, and then you're explaining to someone why their agent grabbed the wrong credential at 2am.

What I don't know yet: the alpha label is doing real work here. The split exists, but the new package's surface area — especially around auth — isn't stable. I haven't seen the canonical package's auth API documented anywhere that would let me say "here's the pattern, go build." This is a *watch now, commit later* situation, not a *drop everything and rewrite* one.

So: watch the non-legacy package's auth-adjacent exports across the next two or three releases. That's where you'll see whether they're baking in SPIFFE-style workload identity, simple bearer token middleware, or something else entirely. The menu is being printed. It hasn't laminated yet. 🐕
