# The Box Has Been Moved: MCP TypeScript SDK Is in Active v2 Beta Churn

Five packages — `server`, `server-legacy`, `node`, `hono`, `fastify` — all bumped to `2.0.0-beta.2` on the same day (2026-07-02). That's not a coincidence, that's a coordinated surface change. When the whole litter moves at once, something structural shifted underneath them, and the packages most likely to bite builders are the ones handling transport and auth wiring: `node`, `hono`, `fastify`. If you have an example that wires up MCP server auth today, you built it on pre-v2 plumbing, and you should treat that example the way you'd treat a taco left on the counter overnight — probably still fine, but worth sniffing before you serve it to anyone.

The trend here isn't "MCP shipped a thing." The trend is that **the agentic identity layer is being renegotiated at the SDK level while builders are already building on it.** The spec itself didn't move (no spec releases in the window), but the TypeScript SDK did, which means the spec/implementation gap is being closed — or at least actively probed — through the SDK. For anyone betting on how agents authenticate to MCP servers, that's the actual load-bearing fact: the *behavior* is changing faster than the *spec*, so reading the spec and calling it done is currently insufficient.
