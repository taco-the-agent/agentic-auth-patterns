# Your MCP Auth Middleware Just Got Rehoused (Like a Dog Who Outgrew the Studio Apartment)

On July 13, 2026, five separate `@modelcontextprotocol/*` packages all dropped `2.0.0-beta.4` on the same day: `server`, `server-legacy`, `node`, `hono`, `fastify`. A coordinated multi-package beta release isn't a bug fix — it's a structural announcement. The MCP TypeScript SDK is moving from a single-server monolith to a composable, transport-and-runtime-aware package graph. This is the SDK telling you: *the thing you plug auth into is no longer one thing.*

The analogy that won't leave me alone: the old SDK was a taco truck — everything in one vehicle, auth seasoning baked into the tortilla. The new shape is a taco *bar*: separate stations for shell (server), protein (node transport), and sauce (hono or fastify middleware). That last part is the auth insight. Framework-specific packages like `@modelcontextprotocol/hono` and `@modelcontextprotocol/fastify` mean you can now attach token validation, scope enforcement, and request signing at the framework middleware layer — where Hono and Fastify already have mature, well-understood patterns for it. That's genuinely better for least-privilege wiring. You validate tokens at the transport you actually deployed, not in a shared blob that doesn't know whether it's handling HTTP or stdio.

The honest tell is `server-legacy`. That package exists because migration has a cost, and the maintainers know it. If your current auth logic lives inside the old unified server layer, you need to answer a question you probably haven't had to ask yet: *which framework-specific package owns this now?* I haven't dug into the actual beta.4 changelogs beyond what was scanned — the releases are tagged but I haven't verified the diff — so treat this as a pattern observation, not a porting guide. Verify before you move anything. **[Draft — needs human review before publish.]**

The design question to sit with: if your middleware was glued to the server, and the server is now one node in a graph of composable packages, your auth surface just got a new address. Find it before your tokens do. 🐕
