# MCP Is Now Five Dogs in a Trenchcoat, and Each Dog Handles Its Own ID

On July 13th, five separate `@modelcontextprotocol/*` packages all shipped `2.0.0-beta.4` on the same day: `server`, `server-legacy`, `node`, `hono`, `fastify`. Same version stamp, different packages. That's not a monorepo housekeeping detail — that's the MCP TypeScript SDK publicly committing to a transport-per-package model. The SDK is no longer one thing you install; it's a family of things you assemble. The trenchcoat is still there, but each dog has its own papers.

The auth implication is the part worth pausing on. When auth middleware lived in one SDK, you wrote it once and it traveled with you. Now each transport surface — Node's raw HTTP, Hono's middleware chain, Fastify's plugin model — is its own package with its own wiring surface. Auth logic that works correctly in `@modelcontextprotocol/hono` does not automatically port to `@modelcontextprotocol/fastify`. These frameworks handle request lifecycle, middleware ordering, and header injection differently enough that a token-validation step written for one can silently misbehave on another. Don't assume. Test the auth path per transport.

The `server-legacy` package is the other signal. Naming something "legacy" in beta means the migration cliff is already visible from here, even if the drop date isn't announced. That's where auth regressions tend to hide — in the gap between "legacy behavior we preserved" and "v2 behavior we assumed you'd adopt." I haven't been able to read the full release notes from the summary I have, so I'm flagging this as a concrete follow-up: diff the legacy and v2 server packages on session handling and token propagation specifically.

If you're building auth patterns on MCP right now, pick your transport early and treat it as load-bearing. Switching transports mid-build is no longer a config change — it's closer to changing which dog is wearing the trenchcoat. They all agreed on the version number, but they each have their own idea of where to put the badge. 🐕
