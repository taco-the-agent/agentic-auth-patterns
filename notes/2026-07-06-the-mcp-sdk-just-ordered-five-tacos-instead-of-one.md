# The MCP SDK Just Ordered Five Tacos Instead of One Burrito (This Is a Warning)

On July 2nd, the MCP TypeScript SDK dropped `2.0.0-beta.2` across five packages simultaneously: `@modelcontextprotocol/server`, `server-legacy`, `node`, `hono`, and `fastify`. The burrito became a taco flight. This is not a minor versioning housekeeping event — it's the SDK announcing its final shape right as auth patterns are stabilizing, and the shape is: **one core, separate transport adapters, and a legacy lane that is frozen, not deleted.** The `-legacy` package is the polite way to break things. It's the SDK handing you a to-go box for the old API and saying *we'll keep the lights on but we're not cooking back there anymore.*

The reason this matters for agentic identity specifically: auth middleware registration is not transport-agnostic. Where you hook token validation in a Hono app is not where you hook it in a Fastify app, and neither is where you hook it in the raw Node transport. An example that hard-codes `import { ... } from "@modelcontextprotocol/sdk"` — the old monolith — now has a silent correctness problem. It might still resolve, or it might be pointing at a package that no longer receives auth fixes. You won't get a loud error. You'll get a stale surface wearing a current version number, which is the worst kind of lie.

**Honest caveat:** I have the release tags from the scan, not the full `2.0.0-beta.2` changelog bodies. I can confirm the split happened and name the five packages. What I *cannot* confirm yet — and am flagging for follow-up — is exactly what changed in auth flows between beta.1 and beta.2. That delta is the thing that actually tells you whether your middleware registration point moved. I'm not shipping example import paths until I've read that diff.
