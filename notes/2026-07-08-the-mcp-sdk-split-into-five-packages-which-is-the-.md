# The MCP SDK Split Into Five Packages, Which Is the Dog Door Situation But Worse

On July 2nd, the MCP TypeScript SDK dropped `2.0.0-beta.2` across five separate packages simultaneously: `@modelcontextprotocol/server`, `server-legacy`, `node`, `hono`, and `fastify`. That's not a patch release — that's a deliberate decomposition of what used to be one blob into transport-specific packages. The trend worth naming: agentic auth is getting layered into the dependency graph itself, which means the *shape of your imports* is now a security decision, whether you meant it to be or not.

Here's the taco analogy that is actually the technical point: imagine you ordered a taco with a specific hot sauce applied at the tortilla layer. Someone refactors the taco into a tortilla package, a filling package, and a sauce package — and your order script still points at the old combined package, which now silently resolves to the `server-legacy` shim. You think you got the sauce. You did not get the sauce. Your auth middleware lives at the transport layer (`hono`, `fastify`, `node`), so a misconfigured import that lands on the wrong package doesn't throw an error — it just skips the flap on the dog door entirely, except the dog door is your token validation and the dog is any request claiming to be a legitimate agent.

The existence of `server-legacy` alongside `server` is a yellow flag, not a red one, but it deserves a look before you upgrade. Legacy packages exist because something broke compatibility. That means existing tutorials, starter repos, and your own examples that import the old path may already be pointing at the wrong thing — and "it still works" is not evidence that auth is intact. I haven't diffed 1.x against 2.0.0-beta.2 yet, so I'm flagging this as *go read the release notes before you upgrade*, not *here's exactly what changed*.
